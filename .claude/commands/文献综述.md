请执行以下步骤完成文献综述生成：

## 0. 论文获取（如果用户提供了论文信息）

如果用户消息中包含 DOI、论文标题或关键词，先通过 Zotero MCP 获取论文：
- DOI → `mcp__zotero__zotero_add_by_doi`
- 标题/关键词 → `mcp__zotero__zotero_search_items`
- 提取元数据 → 写入 `workspace/papers/metadata.json`

## 1. 环境准备

检查并确保：
- `workspace/papers/metadata.json` 中存在论文元数据（若不存在，提示用户使用 `/论文获取` 或手动放入）
- LLM API Key 已配置（`ANTHROPIC_API_KEY` 或 `OPENAI_API_KEY`）
- Python 依赖已安装（`pip install -r requirements.txt`）

## 2. 运行文献导入

```bash
cd "d:\全流程论文写作" && python scripts/run_collect.py --papers-dir workspace/papers
```

## 3. 运行逐结构分析 + 综述合成

```bash
cd "d:\全流程论文写作" && python scripts/run_review.py --backend anthropic --model $ANTHROPIC_MODEL --skip-presentation
```

如果只想分析特定维度（如引言 + 基准结果 + 结论），加 `--sections 1,6,11`。

## 4. humanizer-zh 去AI味

对生成的分析报告调用 humanizer-zh skill 去AI味：

逐个读取 `workspace/analysis/` 下的 `*_analysis.md` 文件，对每个文件：
- 读取内容 → 调用 `Skill(skill="humanizer-zh", args="<文件内容>")` 
- 将去AI味后的文本写回对应文件

同样对 `workspace/literature_review.md` 执行去AI味：
- 读取全文 → 调用 humanizer-zh → 写回

**注意**：去AI味时需要保留学术术语、统计量、表格格式和引用标记不变。

## 5. 检查产出物

确认以下文件已生成：
- `workspace/analysis/` 目录下 11 个维度的分析报告（`*_analysis.md` + `*_analysis.json`）
- `workspace/literature_review.md` — 完整文献综述（已去AI味）
- `workspace/review_analysis_report.json` — 结构化分析报告

## 6. 报告结果

向用户展示产出摘要：论文数量、完成的维度数、综述文件路径、humanizer-zh 处理状态。
